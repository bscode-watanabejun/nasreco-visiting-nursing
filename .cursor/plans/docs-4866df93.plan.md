# ダッシュボード記録ボタンラベル統一

## 現状

ダッシュボードの`VisitRecordButton`では、記録の状態に応じてボタンラベルを切り替えています：

- 記録が完成済み（`completed`）→ 「記録確認」
- 記録が下書き（`draft`）→ 「記録詳細」

一方、スケジュール一覧の`RecordActionButton`では、記録がある場合は常に「記録詳細」と表示しています。

## 修正内容

ダッシュボードの`VisitRecordButton`を修正して、記録がある場合は常に「記録詳細」と表示するように統一します。

### 変更箇所

**ファイル**: `client/src/components/Dashboard.tsx`

**行番号**: 342行目付近

**変更前**:

```tsx
{recordStatus === 'completed' ? '記録確認' : '記録詳細'}
```

**変更後**:

```tsx
記録詳細
```

### 動作

- 記録が完成済み（`completed`）→ 「記録詳細」と表示
- 記録が下書き（`draft`）→ 「記録詳細」と表示
- 記録がない場合 → 「記録」と表示（変更なし）
- ボタンをクリックすると、記録詳細画面に遷移（変更なし）

## 期待される効果

- スケジュール一覧とダッシュボードで表示が統一される
- ユーザーが混乱しにくくなる
