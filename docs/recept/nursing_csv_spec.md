# 訪問看護オンライン請求CSV出力設計書

## 概要

本設計書は、訪問看護療養費のオンライン請求に使用するCSVファイルの出力仕様を定義します。
本仕様は令和6年6月版「オンラインによる請求に係る記録条件仕様(訪問看護用)」に準拠しています。

## ファイル構成

### 全体構造

```
訪問看護ステーション情報レコード (HM)
└─ 訪問看護療養費請求書レコード (GO)
    └─ レセプト情報 (複数)
        ├─ レセプト共通レコード (RE)
        ├─ レセプト情報
        │   ├─ 保険者レコード (HO)
        │   ├─ 公費レコード (KO) [任意・複数可]
        │   ├─ 資格確認レコード (SN) [任意・複数可]
        │   ├─ 受診日等レコード (JD) [任意・複数可]
        │   ├─ 窓口負担額レコード (MF) [任意]
        │   └─ 災害等レコード (GI) [当分の間省略]
        └─ 訪問看護情報
            ├─ 医療機関・保険医情報レコード (IH)
            ├─ 訪問看護指示レコード (HJ)
            ├─ 心身の状態レコード (JS)
            ├─ 傷病名レコード (SY)
            ├─ 利用者情報レコード (RJ)
            ├─ 情報提供レコード (TJ) [任意]
            ├─ 特記事項レコード (TZ) [任意]
            ├─ 専門の研修レコード (KS) [任意]
            ├─ コメントレコード (CO) [任意]
            └─ 訪問看護療養費レコード (KA) [複数]
```

### ファイル形式

- **文字コード**: Shift_JIS
- **改行コード**: CR+LF (Windows形式)
- **ファイル終端**: EOFコード(1バイト)を最終改行の後に記録
- **区切り文字**: カンマ(,)
- **囲み文字**: なし(ダブルクォーテーションなし)

## レコード仕様

### 共通ルール

1. **レコード識別**: 各レコードは2文字のアルファベットで識別
2. **モード区分**:
   - 英数: 半角英数字
   - 数字: 半角数字のみ
   - 漢字: 全角文字
3. **項目形式**:
   - 固定: 指定バイト数を必ず使用
   - 可変: 最大バイト数以内で可変長
4. **省略**: 可変項目で記録不要な場合は空欄(カンマ間に何も記録しない)

## データ検証ルール

### 必須項目チェック

1. **訪問看護ステーション情報レコード (HM)**
   - 全項目必須(電話番号除く)

2. **レセプト共通レコード (RE)**
   - レセプト番号、レセプト種別、請求年月、氏名、男女区分、生年月日は必須

3. **保険者レコード (HO)**
   - 保険者番号、被保険者証番号は必須

4. **訪問看護療養費レコード (KA)**
   - 算定年月日、負担区分、療養費コードは必須

### 整合性チェック

1. **日付の妥当性**
   - 生年月日 < 訪問開始年月日 < 算定年月日
   - 指示期間自 < 指示期間至

2. **コードの妥当性**
   - 各種コードが対応する別表の範囲内

3. **レコード順序**
   - ファイル構成に従った順序
   - 資格確認レコード、受診日等レコードは負担者種別コード昇順

4. **職種等コードの一貫性**
   - 同日訪問回数が"02"または"03"の場合、職種等コードが訪問回に対応

## 実装における注意事項

### 1. 文字エンコーディング

- Shift_JISでの出力を徹底
- 機種依存文字の使用に注意
- カナは全角カタカナ、長音符は"ー"(815B)を使用

### 2. 日付フォーマット

- 年月日: YYYYMMDD (例: 20250615)
- 年月: YYYYMM (例: 202506)
- 時刻: HHMM (例: 1430)

### 3. 数値のゼロ埋め

- 固定長フィールドは必ず指定桁数で出力
- 可変長フィールドは先頭ゼロ不要(ただし枝番など一部例外あり)

### 4. 空白項目の扱い

- 可変長で記録不要な場合は何も記録しない(カンマ連続)
- 固定長は必ず指定バイト数を出力

### 5. 改行とEOF

- 各レコード末尾に改行(CR+LF)
- ファイル最終行の改行後にEOFコード(0x1A)を1バイト追加

### 6. レセプト種別の判定

- 保険区分と給付割合から自動判定
- 医保単独、公費併用、公費単独、後期単独などを正しく設定

### 7. 資格確認レコードの記録条件

- 訪問看護ステーションからの請求時は必ず記録
- 保険者レコード記録時は負担者種別"1"
- 公費レコード(12始まり)記録時は負担者種別"2"~"5"

### 8. 同日複数回訪問時の職種等

- 同日訪問回数に応じて職種等コードの列を使い分け
- 例: 1回目は"03"、2回目は"13"

## 付録: よくある質問

**Q1: 枝番は必須ですか?**
A1: 電子資格確認を行った場合または被保険者証に記載がある場合は記録します。記載がない場合は省略可能です。

**Q2: 再入院等区分はどう記録しますか?**
A2: 令和6年改定により、複数回再入院があった場合は特記事項レコード(TZ)に記録変更となりました。

**Q3: GAF尺度は必須ですか?**
A3: 精神科訪問看護基本療養費(Ⅰ)または(Ⅲ)を算定した場合のみ、心身の状態レコード(JS)に記録します。

**Q4: 同日に4回以上訪問した場合の職種等コードは?**
A4: 3回目以降は任意の訪問回の職種を"3回目以降"のコード(21-30, 71-80)で記録します。
