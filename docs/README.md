## 📝 ドキュメント作成のガイドライン

### 新規ドキュメント作成時

1. **ファイル名**: 日本語OK、わかりやすい名前をつける
2. **先頭に概要**: 実装日、対象、目的を明記
3. **見出し構造**: 階層的に整理（#, ##, ###）
4. **コードブロック**: 言語指定を忘れずに
5. **表の活用**: 比較や一覧は表形式で
6. **実装例**: 具体的なコード例を含める

### 更新時

- ファイル先頭の「最終更新」日付を更新
- 変更履歴セクションがあれば追記
- 古い情報は削除せず、「非推奨」などの注記を追加
